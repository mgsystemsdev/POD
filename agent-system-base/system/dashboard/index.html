agents CLI — Command reference

Quick reference for the agents toolchain. Behavior lands as each subcommand is implemented; some flags are dry-run only until wired.

agents init

Scaffold .claude/ (pipeline, governance, specialists) and project wiring files without nuking existing content (when implemented as discussed).

agents add

Register the current repo as a project in the dashboard/API and capture linkage details for this workspace.

agents register

Register or re-point a project by slug, name, or path from anywhere (not tied to cwd).

agents pull

Download authoritative Railway state into the local 16-file cache (overwrite local cache with cloud truth).

agents pull --dry-run

Show what pull would fetch or overwrite, without writing files.

agents push

Upload local cache changes back to Railway/DB for the tabs that are push-supported.

agents push --dry-run

Show what push would send, without writing to the database.

agents tasks

Import or normalize tasks.json into the task system (escape hatch when tasks are wedged).

agents dashboard

Start the local dashboard server for instant UI inspection.

agents dash

Shortcut for agents dashboard.

agents stop

Stop the local dashboard server.

agents status

One-screen health and readiness: config, connectivity, last pull/push markers, and push prerequisites.

agents doctor

Alias of agents status (same output, easier to remember).

agents open

Open or print the right dashboard URL for this project from api_url + project_id.

agents help

Print the command list and basic usage.
