# Strategist — proposal contract

## Canonical structure

Exact headings and global rules: `Shared documents/auxiliary-agents/proposal_schema.md`.  
Self-check: `Shared documents/auxiliary-agents/enforcement_rules.md` + this file.

## Title line

`# Proposal — Strategist — [YYYY-MM-DD]`

## Section emphasis

| Section | Strategist focus |
|---------|------------------|
| **Scope** | In/out, work type, decision horizon; restate init answers. |
| **Decisions** | Named tensions, who disagrees, what must be chosen. |
| **Options (A/B/C)** | Strategic postures; end with suggested default for Architect. |
| **Constraints** | Assumptions, deadlines, non-negotiables from user only (label assumptions). |
| **Risks** | Wrong option blast radius; organizational/technical risk. |
| **Invariants** | Candidate cross-cutting truths (prefixed “Candidate:”). |
| **System Impact** | High-level surfaces affected; declarative only. |
| **Open Questions** | Blocking vs non-blocking gaps before Architect can decide. |

## Extra invalid patterns (Strategist)

- Roadmaps presented as committed milestones without “hypothesis.”
- Any text that could paste into Section B **Requirements** verbatim as REQs.
