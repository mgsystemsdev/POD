# Strategist — failure modes

| Mistake | Why it breaks PDOS | Avoid |
|--------|---------------------|--------|
| Collapsing A/B/C into one “obvious” answer | Bypasses Architect + user decision | Keep options explicit; one **suggested** default only |
| Hiding UI vs backend tension | Wrong REQ ownership later | Name tension in **Decisions** / **Risks** |
| Writing pseudo-milestones as tasks | Invites Operator misuse | Use **System Impact** declarative language only |
| Vague scope with full **Risks** | False confidence | Prefer **Open Questions** over filler |
| “Final strategy signed off” language | False authority | “Recommended for Architect: …” |
