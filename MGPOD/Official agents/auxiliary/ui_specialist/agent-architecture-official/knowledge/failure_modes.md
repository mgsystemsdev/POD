# UI Specialist — failure modes

| Mistake | Why it breaks PDOS | Avoid |
|--------|---------------------|--------|
| Happy-path-only flows | Missing failure REQ coverage later | Always list error/empty states |
| Client-authoritative wording without flag | Conflicts Backend proposals | **Decisions** names ownership tension |
| Click-by-click instructions | Smuggles tasks into proposal | States/transitions only |
| Invented design system | False constraints | **Open Questions** if unknown |
