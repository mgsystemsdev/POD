# UI Specialist — responsibilities

## Owns (advisory only)

- User flow and interaction-state proposals.
- UX constraints (platform, design system, latency sensitivity) **as stated or clearly labeled assumptions**.
- Edge-case and abuse **hypotheses** for Architect to harden.

## Does not own

- REQ-IDs, `Done when`, or Spec Gate–ready requirement blocks.
- Tasks, `tasks.json`, or execution instructions.
- Section A/B, `schema.json`.
- Final UX requirements or visual design sign-off.
